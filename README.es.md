# Frontend Mentor – Solución Landing Page Clipboard

Esta es mi solución al desafío **Clipboard Landing Page** de Frontend Mentor. El proyecto se centra en la construcción de una landing page limpia, moderna y completamente responsive, basada en un diseño proporcionado, siguiendo buenas prácticas de HTML semántico, CSS responsive y una arquitectura de estilos escalable.

Este desafío fue una excelente oportunidad para practicar desarrollo mobile-first, layouts con Flexbox y Grid, propiedades personalizadas de CSS y microinteracciones de interfaz.

## Descripción general
Este proyecto es una landing page de producto responsive que se adapta perfectamente a dispositivos móviles, tablets y escritorio. Incluye una sección hero con botones call to action, múltiples secciones de contenido que explican las funcionalidades del producto, logos de marcas y un footer estructurado con navegación y enlaces a redes sociales.

El objetivo principal fue replicar fielmente el diseño original de Frontend Mentor, manteniendo una base de código limpia, semántica, accesible y fácil de mantener.

El proyecto sigue un enfoque mobile-first y mejora progresivamente el layout para pantallas más grandes utilizando técnicas modernas de CSS.

---

## El desafío
Los usuarios deben poder:

- Ver el layout óptimo según el tamaño de pantalla de su dispositivo.
- Experimentar un diseño completamente responsive en móvil, tablet y escritorio.
- Visualizar estados hover en todos los elementos interactivos como botones, enlaces e iconos sociales.
- Navegar por un layout que se aproxime lo más posible al diseño original de Clipboard.
- Interactuar con una interfaz limpia y accesible construida con HTML semántico.

## Proceso de desarrollo
- Estructuré la página utilizando elementos **HTML5 semánticos** como `main`, `section`, `article`, `nav` y `footer`.
- Apliqué un flujo de trabajo **mobile-first**, diseñando primero para pantallas pequeñas y escalando con media queries.
- Construí los layouts utilizando **Flexbox** para alineación y estructura de componentes, y **CSS Grid** para secciones más complejas.
- Usé **CSS custom properties (variables)** para gestionar los colores y mantener un sistema de diseño consistente.
- Seguí la metodología **BEM (Block Element Modifier)** para el nombrado de clases, mejorando la legibilidad y escalabilidad del CSS.
- Implementé imágenes de fondo responsivas para distintos tamaños de pantalla.
- Añadí estados hover y transiciones en botones, enlaces e iconos SVG para mejorar la experiencia de usuario.
